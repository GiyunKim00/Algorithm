# 덱을 이용한 구간 최댓값 트릭
## 개요
- 특정 구간 내에서의 데이터들을 우선순위가 높은 값으로 정렬하는 알고리즘
## 방식
- 덱에는 항상 우선순위 높은 값 후보들만 내림차순으로 유지함.
- 값이 아니라 인덱스를 저장하여 구간에서 벗어났는지 확인함.
- 덱의 불변식
  - 인덱스는 덱의 앞 → 뒤로 갈수록 증가
  - 값은 앞 → 뒤로 갈수록 감소
  - 구간에서 벗어난 원소는 앞에서부터 제거
  - 우선순위가 기존보다 낮아 쓸모없는 후보는 뒤에서부터 제거
    - 예 : 1 3 4 에 2가 들어오면? 3과 4는 어떤 경우에도 구간 내에서 우선순위가 높을 수 없음. 따라서 제거 -> 1, 2
## 사용 이유
- 시간복잡도가 O(N)임. 각 원소는 덱에 최대 1번 들어가고, 최대 1번 나옴.
- 브루트포스는 구간 내에서 모든 원소를 다 검사하므로 O(NK)임.
- 우선순위 큐는 브루트포스보다는 성능이 낫고 덱 트릭보다는 구현이 쉬우나 O(N log N)임.
- 세그먼트 트리는 전처리와 처리를 합쳐 O(N log N)이며 여러 쿼리에 좋으나 슬라이딩 윈도우에는 과함.
## 언제 쓰면 좋은가?
- 연속된 K개
- 각 구간의 최댓값 / 최솟값
- 슬라이딩 윈도우
- N이 클 떼 (10⁵ ~ 10⁶)
